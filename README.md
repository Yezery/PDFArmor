# PDF 加密工具

> 一个**纯浏览器运行**的 PDF 加密工具：批量分发双密码、7 档管理员权限细分、AES-256 全本地运算，文件不上传任何服务器。

双击 `index.html` 直接使用。算法采用 PDF 2.0 **V=5 R=5 / AES-256**，Adobe Reader X 以后所有主流 PDF 阅读器兼容。

---

## 功能

- **批量加密**：CSV 密码表驱动，一行一个文件名对应一个密码，适合一次处理几十到几百份 PDF。
- **单文件加密**：拖入一个 PDF + 输入密码 → 直接下载加密版本。
- **7 档权限细分**：覆盖 PDF 标准的 R3/R4/R5/R6/R9/R11/R12 全部 7 个权限位，可单独控制打印（低 / 高分辨率）、修改、复制、注释、表单、文档组装。
- **双密码机制**：用户密码打开 PDF、管理员密码解除权限限制，两者独立，必须不同。
- **完全本地**：PDF 文件从未离开浏览器进程。加密库第一次从 jsDelivr CDN 加载，加载完成后可断网使用。

---

## 两种模式

### 批量

适合多人分发场景：HR 发合同、销售发报价、法务发敏感文档归档。

流程：选文件夹 → 导出 CSV 模板 → 填表 → 上传 → 点加密 → 自动打包 zip 下载。

### 单文件

临时加密一两个 PDF 不用准备 CSV：选文件 + 输密码 + 加密。下载文件名自动加 `_encrypted` 后缀。

两 tab 的权限**独立配置**——批量模板和临时加密各管各的，互不干扰。

## 权限细分

默认全开（PDF reader 不限制任何操作）。每取消一个勾选 = 禁止该操作。

7 个复选框对应 PDF 标准的 R 位：

| UI 项 | R 位 | 位值 |
|---|---|---|
| 🖨 打印（低分辨率） | R3 | 0x008 |
| 🖨 打印（高分辨率） | R12 | 0x1000（含 R3） |
| ✏️ 修改文档内容 | R4 | 0x010 |
| 📋 复制 / 提取内容 | R5 | 0x020 |
| 💬 添加 / 修改注释 | R6 | 0x040 |
| 📝 填写表单 | R9 | 0x200 |
| 🔧 文档组装 | R11 | 0x800 |

写进 PDF 的 P 字段按 spec 编码，Adobe Reader / 福昕 / WPS 严格遵守。Chrome / Edge 内置 viewer 读取没问题，权限执行得看具体 reader 行为。

## 加密算法

- **内容加密**：AES-256-CBC，每对象随机 IV，走 `crypto.subtle`（浏览器原生，含硬件 AES-NI 加速）
- **密钥派生**：SHA-256（asmcrypto.js，纯 JS 高性能实现）+ AES-128-ECB（qpdf 10.x 算法 8/9/10）
- **完整性**：`/Perms` 字段校验打开后是否被改过
- **元数据**：`/EncryptMetadata = true`，保留 PDF 元数据（搜索 / 书签可用）

> 标准合规：pikepdf v10.9.1 交叉校验生成结果，V=5 R=5 / CFM=AESV3 / OE=32B / UE=32B / Perms=16B 全部通过。

彻底告别 RC4 和 MD5。

## 安全性

**PDF 文件不上传**：拖进去的二进制在浏览器 JS 里加解密，产物直接走 `Blob` 下载，整条链路无任何 fetch 上传。可以断网验证：第一次加载 CDN 后拔网线，依旧能继续加密。

**AES-256 设计上无法绕过**：忘了用户密码救不回——这是 PDF 规范的特性，不是工具的 bug。管理员密码能解除权限限制，但**看内容仍要用户密码**。

**两个密码不能相同**：管理员密码泄露意味着权限失效。默认 `Admin@2026`，建议改。

## CSV 格式

```csv
name,password
张三的合同.pdf,123456
李四的合同.pdf,654321
王五的合同.pdf,
```

- 第一行必须是表头 `name,password`（不区分大小写）
- `password` 留空 = **跳过这个文件**，不报错
- 支持中文、emoji、`.xlsx` / `.xls`
- 文件名必须跟 PDF 完全匹配——直接用工具导出的模板开始填，最不容易出错
